surface.py refactor · ledctl

From spatial graph to VJ vocabulary.

The current control surface is a clean, typed graph of scalar_field × scalar_t × palette → rgb_field — and the LLM uses it beautifully. What it can't say yet is "around the loop", "shoot from centre", "flip every 4 beats", "ripple on kick". This plan adds those words without doubling the system-prompt budget.

TL;DR. Keep the {kind, params} tree paradigm. Add (1) named topology frames like u_loop, axial_dist, side_signed so motion can address the rectangle's perimeter and centre. (2) Transient / particle leaves — comet, ripple, bouncing, geq_bars, starburst, fire. (3) Beat-aware time — bpm_clock, beat_count, audio_peak, step_select. (4) Recipes — compound effects exposed as one node so the LLM and operators get expressive shorthand without a token-prompt explosion. Token budget projected 2.5k → 6.2k. Roughly 14–16 days, phased so the system stays working after every step.

1What we keep · what's missing

The compiler, the strict per-primitive pydantic validation, the polymorphic combinators, the live generate_docs() that feeds both the system prompt and GET /surface/primitives — all load-bearing, all preserved. Same with pulse(by, floor), the HSV palette baking, and the numeric / palette-string sugar in node slots. The refactor is purely additive on vocabulary; nothing in engine.py / mixer.py / masters.py needs to change behaviourally.

The five gaps the festival-day shows up in:

No path-aware spatial frames

Only Cartesian x/y/z + radial-from-point. The rig is a perimeter rectangle; "around the loop" requires manual top/bottom glue.

No transient / event primitives

No comet, ripple, meteor, bouncing balls, GEQ bars, starburst — the moves that earn WLED its expressiveness budget.

No beat-aware time

Only wall-clock LFO + smoothed audio bands. Can't say "every 4 beats", "trigger on kick peak", "increment a counter on snare".

Limited spatial → spatial transforms

No spatial blur, mirror, kaleido, displace, region-mask, integer pixel-shift. WLED uses these constantly.

Token economy not yet a design pressure

Adding 50 leaf primitives the WLED way explodes the prompt. We need recipes on top of an expanded leaf set.

Three meta-patterns the surface is missing words for:

Emitter + integrator + fade buffer

Comet / meteor / ripple / bouncing / starburst are all the same shape: a small list of (pos, vel, ttl, hue) structs, an update(dt), a render() into a fade buffer.

Peak triggers

A boolean impulse on transient detection. Lets emitters fire on the music, not the clock — the difference between "audio-reactive" and "audio-driven".

Indexed sines (BPM-locked)

Half of WLED's "magic" is N independent beatsin8(speed*i) positions, each a different colour. Lock them to BPM and they breathe with the room.

3Topology frames — the centre of gravity

Today the agent only has Cartesian x / y / z. To prompt "clockwise around the rectangle" the LLM has to manually compose wave(x) * mask(y > 0) + reverse_wave(x) * mask(y < 0). It doesn't. The fix is to expose named coordinate frames — derived once at topology build time, indexed by name.

The rig — verified 2026-05-08. Spatially two parallel 30 m rows 1 m apart, all four strips centre-fed. Conceptually, a perimeter loop the agent should be able to address by name.

4Expanded primitive families

Each new leaf fills a gap WLED has shown is musically valuable, has a small Params (≤ 5 numeric/discrete fields), and a one-line summary in the registered docs. Compound multi-step recipes go in §5 — this section is the leaves they compose from.

scalar_t — clock + audio

bpm_clock

BPM-synced sawtooth in [0, 1) per beat. Divisor 1/2/4/8 = beat / half / quarter / eighth.

beat_count

Floor of wall_t · bpm / 60 mod N. The integer index for step_select.

audio_peak

Schmitt trigger: 1.0 for hold_s after rising edge above threshold, else 0. The kick detector.

audio_envelope

Per-band attack/release if the audio server's smoothing is wrong for the visual.

audio_band_n

Direct read of bin 0..15 from the 16-bin GEQ. Underlies geq_bars.

gravity_meter

Fast attack / linear decay. The mode_gravcenter top-LED state shape.

latch

Sample-and-hold: hold input while gate is high, release on falling edge.

random_each

New PRNG draw every period_s (or every gate rising edge).

slew

Rate-limit a scalar — max change up/down per second.

step_select

Pick element i from a list. The direction-flip primitive.

counter

Increments on rising edge of trigger. Use with step_select to advance through states.

scalar_field — spatial generators

frame

Replaces position. Reads any registered topology frame by name.

multi_wave

Sum of 2–3 phase-offset waves. Plasma / lake collapsed to one node.

radial_loop

Radial but along the perimeter — distance is u_loop arc length, not Cartesian.

pulse_train

N evenly-spaced gaussian bumps moving along an axis. "Two dots", "four dots", "eight-segment chase" in one node.

lighthouse

Rotating beam — narrow gaussian indexed by angle minus phase.

gradient_mask

Piecewise linear ramp with N stops on an axis. Region-masking workhorse.

voronoi

Nearest-of-N moving seeds → cell-ID field. Cellular shapes.

rgb_field — transient / particle leaves

Each owns a fixed-size particle pool, integrates per frame, draws into an internal fade buffer. Determinism guaranteed by seed.

comet · comets

One head + exp. fading tail, or N phase-offset comets along an axis.

meteor

Like comet but stochastic per-pixel decay (the WLED meteor shape).

ripple

Outward propagating bumps. Auto-emits on trigger rising edge. Combine with audio_peak for kick-driven ripples.

bouncing

Gravity-integrated balls along an axis. Bounce loss + random impact restart.

geq_bars

16-bar spectrum analyser along any axis. Mirror, peak hold, palette.

starburst

Emit a fragment burst on trigger. Fragments fly out and fade.

lightning

Multi-flash strobe envelope on trigger. flashes, decay_ms.

fire

1-D heat lattice along an axis (mode_fire_2012). Cooling / sparking.

chase_dots

M evenly-spaced dots scrolling — clean, no fade, the WLED-style chase.

combinators — spatial transforms

blur_axis

Box-blur a scalar/rgb field along one axis. Hides the centre seam.

mirror

Fold a field along an axis (x → |x| then re-stretch).

kaleido

N-way fold (replicate + mirror). Pair with u_loop for "8-fold around".

displace

Sample input field at offset positions modulated by another scalar field. Turbulence.

where

Per-LED ternary: mask >= 0.5 ? a : b. Finally fixes the "use mix.t to split top/bottom" anti-pattern.

region_mask

Pre-baked masks: top, bottom, left, right, loop_quarter[0..3].

tile

Repeat a field N times along an axis. 1-cycle wave → tiled.

pixel_shift

Integer-pixel chase along an axis. Crisp on dense LEDs; differs from continuous wave.

palette — new helpers

palette_dynamic_hue

Palette whose hue stops are themselves scalar_t nodes — beat-driven colour cycle in one node.

palette_window

Shrinks/centres the LUT range used. Scroll a segment of rainbow with audio-driven width.

5Recipes — compound effects, one node

This is the token-economy lever. A recipe is a primitive whose compile() returns a sub-tree of leaves the agent never sees. From the LLM's view: one node with 4–6 params. From the renderer's view: a normal compiled subtree.

What the LLM writes

{ "kind": "loop_orbit", "params": { "count": 3, "speed": 0.5, "palette": "rainbow", "flip_every_beats": 8, "band": "low", "floor": 0.4 } }

One node · ~30 output tokens · "clockwise rainbow chase that flips every 8 beats and pulses on kick".

→

What the recipe compiles to

palette_lookup(
  palette: "rainbow",
  scalar: wave(
    axis: "u_loop",
    wavelength: 0.333,
    shape: "sawtooth",
    speed: mul(
      a: 0.5,
      b: step_select(
        index: beat_count(divisor: 8, mod_n: 2),
        values: [+1.0, -1.0]
      )
    )
  ),
  brightness: pulse(
    by: audio_band(band: "low"),
    floor: 0.4
  )
)

Why recipes (and not just more leaves)

Token economy

Recipe ≈ 80 prompt tokens. Equivalent low-level subtree the LLM has to write every turn ≈ 250-400 output tokens, 6-8 nodes for the type-checker — and the LLM still gets the topology wrong half the time.

Self-correction is cheaper

Recipes have validated invariants (palette is always palette, axis is always a frame name). Most "layer_validation_failed" loops we see today are LLMs over-nesting; recipes flatten the path.

Operator presets get small

config/presets/peak.yaml becomes ~5 lines instead of 40.

Inspectable, not opaque

GET /surface/expand/{kind}?params=… returns the expanded NodeSpec. Operator UI gets a "show as tree" button — drill in, tweak one wave inside.

6Audio + beat layer

scalar_t
audio_peak (band, threshold, hold_s, cooldown_s)

Schmitt trigger on band crossing threshold. Returns 1.0 for hold_s after rising edge, then 0. cooldown_s blanks re-triggers. The LLM-visible kick detector.

scalar_t
audio_envelope (band, attack_s, release_s)

Override the audio server's smoothing for the rare "hard-attack envelope on the snare" prompt.

scalar_t
audio_band_n (bin: 0..15)

Direct read of the 16-bin GEQ from the audio server. Soft-fail to 0 if bands aren't in the OSC packet.

scalar_t
bpm_clock + beat_count

BPM-synced. New MasterControls.bpm master + tap-tempo button. Phase 2: auto-detect from the audio server.

Budget knobs we keep available:

  1. Tag building-block primitives audience: "operator". Visible in /surface/primitives for the UI; hidden from generate_docs(). Saves ~1k tokens.
  2. One-line descriptions. Pulse + Sparkles summaries are 5 / 7 lines today; one-line is the rule.
  3. Trim anchor examples from 7 → 4: one recipe, one custom tree, one peak-driven, one beat-direction-flip.
  4. Compact param tables. Drop full descriptions in the prompt (operator UI gets the full schema via primitives_json anyway).
CI guard. assert len(tokenize(generate_docs())) < 8000 in CI so prompt bloat is caught early. Snapshot test on system_prompt.build(...) against a representative install for agent regressions.

Recipes share machinery via a Recipe base class:

class Recipe(Primitive):
    """A primitive that compiles to a sub-tree of leaf primitives."""

    @classmethod
    def expand(cls, params: BaseModel, topology: Topology) -> NodeSpec:
        raise NotImplementedError

    @classmethod
    def compile(cls, params, topology, compiler):
        sub = cls.expand(params, topology)
        return compiler.compile_child(sub, expect=cls.output_kind, path="(recipe)")

12What we deliberately are not doing

  • No multi-tool agent loop. One update_leds per turn stays.
  • No client-side rendering / GPU. All numpy on the Pi. Particle pools bounded so worst-case per-frame cost is provable.
  • No 2D matrix mode. WLED's mode_2D* stays out — we have no matrix; u_loop already gives the visually-2D moves.
  • No sequencer. Mixer crossfade + presets cover scene transitions; preset save/recall is the sequencer.
  • No surface-side LFO sync. Two lfo nodes run independent clocks; bpm_clock phase-locks globally to the master BPM — the only sync the agent should think about.
  • No JS-style scripting in params. Every parameter is a node or a literal. The graph stays inspectable, type-checked, snapshotable, 100% deterministic given (seed, t).

surface.py already nailed the hard part: a typed, composable graph the LLM can write into. What's left is vocabulary — the rectangle, the loop, the beat, the comet. Add path-aware frames, a particle-leaf family, beat-aware time, and 13 recipes; split the file before it gets unmanageable; keep the prompt under 8 k tokens; and the same agent that today produces "warm drift" can produce "fire fountains exploding from centre on every kick, with a slow ice ripple drifting clockwise around the loop". That's the festival.
